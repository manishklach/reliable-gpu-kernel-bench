GPU Run Checklist

This checklist is the shortest path from the current CPU-backed prototype to a real Triton-backed accelerator run.

1. Copy the folder

Copy the entire folder below to the GPU machine:

C:\Users\ManishKL\Documents\Playground\kernel_noise_prototype

The Triton tutorial source files are already inside triton_tutorials.

2. Prepare the Python environment

Install a CUDA-capable PyTorch build and Triton in the same environment. Then install the project dependencies:

python -m pip install -r C:\path\to\kernel_noise_prototype\requirements-gpu.txt

Important This must be a GPU-enabled environment. CPU-only PyTorch will not run the Triton paths.

3. Preflight check

Run the built-in preflight script:

python C:\path\to\kernel_noise_prototype\setup_gpu.py

Expected checks:

  • torch.cuda.is_available() == True
  • Triton importable == True
  • Matmul tutorial exists and imports
  • Softmax tutorial exists and imports
  • nvidia-smi is visible and returns GPU information

Pass condition Both tutorial imports succeed and CUDA is available.

4. Run the real Triton workloads

Fastest path:

powershell -ExecutionPolicy Bypass -File .\run_gpu_flow.ps1

This runs preflight, single-run Triton matmul, generated matmul, the Triton batch runner, and refreshes the variant search report.

Matrix multiplication workload:

python C:\path\to\kernel_noise_prototype\demo.py --backend triton --workload matmul

Generated matmul variant workload:

python C:\path\to\kernel_noise_prototype\demo.py --backend triton --workload matmul --candidate-mode generated

Repeated generated-variant batch run:

python C:\path\to\kernel_noise_prototype\triton_batch_runner.py --runs 12 --workload matmul --candidate-mode generated

Fused softmax workload:

python C:\path\to\kernel_noise_prototype\demo.py --backend triton --workload softmax

5. Inspect outputs

After each run, inspect:

  • latest_run.html
  • latest_run.json
  • latest_run.txt
  • batch_outputs/triton_matmul_generated/triton_batch_summary.html

These show the naive winner, finalist-pair analysis, final decision, and contamination rates.

6. Suggested first GPU validation sequence

  • Run setup_gpu.py
  • Run Triton matmul once and verify reports are generated
  • Run Triton softmax once and verify reports are generated
  • Capture one or two screenshots of latest_run.html and terminal output for internal sharing
  • If telemetry is stable, consider adding a Triton batch simulator next

7. Known constraints

  • The current multi-run simulation is CPU-backed and synthetic; it demonstrates the decision logic, not real GPU throughput.
  • The Triton adapter trims the tutorial files before their top-level demo code, so importing them should not auto-run tutorial plots.
  • If a GPU machine has different tutorial paths, set TRITON_TUTORIAL_MATMUL or TRITON_TUTORIAL_SOFTMAX.
