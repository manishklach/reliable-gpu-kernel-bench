Technical Specification: Noise-Aware Kernel Benchmarking Prototype

Workspace path: C:\Users\ManishKL\Documents\Playground\kernel_noise_prototype

Purpose. This prototype implements the patent-centered control loop: telemetry-derived contamination detection, pairwise ambiguity-guided rerun allocation, and confidence-gated promotion for generated kernels or kernel-like candidates.

1. Scope

The prototype is intentionally split into two layers. The first layer is the selection engine in engine.py, which contains the contamination classifier, finalist ranking, pairwise ambiguity analysis, rerun budget logic, and promotion gate. The second layer is the workload adapter layer, which currently includes a CPU-backed simulation path in demo.py and a GPU-ready Triton adapter in triton_adapter.py.

Current runnable modes

  • python demo.py --backend cpu
  • python simulate_results.py --runs 50

Prepared GPU modes

  • python demo.py --backend triton --workload matmul
  • python demo.py --backend triton --workload softmax

4. Telemetry and Contamination Model

The contamination classifier evaluates telemetry per trial. The current implementation uses CPU load, thermal delta, SM-clock drop, memory-clock drop, active-process count, launch CPU affinity, NUMA placement, and an explicit synthetic interference flag. These signals are chosen to align with the patent thesis that contamination is defined by telemetry-derived hardware and host execution state, not by timing variance alone.

Signal Current rule Effect
SM clock Contaminated if drop exceeds 5% Adds sm_clock_drop reason
Memory clock Contaminated if drop exceeds 5% Adds mem_clock_drop reason
Thermal delta Contaminated if increase exceeds 7 C Adds thermal_drift reason
CPU load Contaminated if above 95% Adds cpu_load_high reason
Concurrent activity Contaminated if active processes > 1 Adds concurrent_activity reason

5. Pairwise Ranking and Promotion Logic

Finalist ranking

Finalists are not selected by raw median. The engine computes a contamination-penalized ranking from acceptable-trial medians only. Candidates with weak acceptable-trial evidence receive an evidence penalty before entering the final pair.

Promotion gate

The current gate requires at least 10 acceptable trials for both finalists, at least 2% measured gain, at least 1.5% lower-confidence-bound gain, winner contamination at or below 15%, and loser contamination not more than 2x that threshold.

6. Small Generated-Kernel Layer

The matmul Triton path now supports a small generated-candidate mode. This does not synthesize arbitrary CUDA/PTX, but it does instantiate multiple real kernel candidates from a structured search space derived from the official Triton matmul tutorial. Each candidate fixes a specific tuple of block sizes, group size, number of warps, and number of stages, then routes that candidate through the same contamination-aware benchmark engine.
